Run Python in a browser-harness-compatible, raw-CDP browser environment.

This is the browser interaction tool. Use other tools for files, commands, plans, helper sessions, and completion; use this tool for browser navigation, browser inspection, clicks, typing, screenshots, downloads, uploads, network inspection, extraction, and browser-backed verification.

Browser work is shared mutable state, so keep browser actions sequential and deliberate. Use parallel local tools for independent read-only file or command inspection, but do not parallelize Python browser actions with other browser-changing tools.

CDP is the source of truth. Use raw CDP for basic browser control: `cdp("Page.navigate", url="https://...")`, `cdp("Runtime.evaluate", expression="document.title", returnByValue=True)`, `cdp("Input.dispatchMouseEvent", type="mousePressed", x=..., y=..., button="left", clickCount=1)`, `cdp("Input.dispatchMouseEvent", type="mouseReleased", x=..., y=..., button="left", clickCount=1)`, `cdp("Input.insertText", text="...")`, and `cdp("Page.captureScreenshot", format="png")`. Helpers are conveniences for host behavior, not a framework: `new_tab`, `goto_url`, `page_info`, `navigation_snapshot`, `embedded_data_snapshot`, `repeated_items_snapshot`, `extract_repeated_items`, `rows_snapshot`, `extract_grid_rows`, `form_fields_snapshot`, `fill_form_field`, `action_controls_snapshot`, `click_button`, `overlay_actions_snapshot`, `dismiss_overlay`, `form_controls_snapshot`, `toggle_form_control`, `select_controls_snapshot`, `select_option`, `read_document_text`, `arxiv_query`, `js`, `browser_fetch`, `browser_fetch_many`, `capture_screenshot`, `screenshot`, `screenshot_clip`, `emit_image`, `click_at_xy`, `fill_input`, `type_text`, `press_key`, `scroll`, `wait_for_load`, `wait_for_element`, `wait_for_network_idle`, `current_tab`, `list_tabs`, `switch_tab`, `ensure_real_tab`, `upload_file`, `drain_events`, `http_get`, `http_get_many`, `copy_artifact`, `artifact_root`, `outputs_dir`, `audit_artifact`, `agent_workspace`, `load_agent_helpers`, `domain_skills_for_url`, `last_domain_skills`.

Do not import Playwright, Selenium, or Pyppeteer. Browser-harness workflow: first navigation should usually be `new_tab(url)`, not `goto_url(url)`, because `goto_url` mutates the active tab. If the task is site-specific and `domain_skills_for_url(url, include_content=True)` returns files, read those domain skills before inventing selectors, private API routes, or flows. Use screenshots as labeled temporal checkpoints: initial load, before/after meaningful clicks, scrolls, route changes, menus, dialogs, downloads, uploads, form submissions, and final verification. Prefer coordinate clicks for visible targets: `capture_screenshot` or `screenshot`, inspect the pixels, `click_at_xy(x, y)`, then screenshot again. Prefer capturing the action timeline inside one Python call when possible. Do not call `screenshot` repeatedly on an unchanged viewport; after a screenshot, act, inspect with CDP/JS, navigate, scroll, call `screenshot_clip(...)` for a different CSS-pixel region, wait for an async transition, or finish. For cookie/privacy/modal overlays, call `overlay_actions_snapshot()` if needed and click with `dismiss_overlay(...)` before extracting. For labeled/split/autocomplete forms, call `form_fields_snapshot()` and fill with `fill_form_field(...)` before coordinate guessing; for named Search/Submit/Apply/Next/Download/Save controls, call `action_controls_snapshot()` if needed and click with `click_button(...)`; for labeled checkboxes/radios/switches, call `form_controls_snapshot()` if needed and set state with `toggle_form_control(...)`; for native selects and combobox-like dropdowns, call `select_controls_snapshot()` if needed and choose with `select_option(...)`. Before deciding a site lacks a property/listing/document/investor/results page, call `navigation_snapshot(keywords=[...])`; if it shows a collapsed menu/toggle control, click it, wait, and call `navigation_snapshot` again. On ecommerce, directory, product, investor/document, and SPA listing pages, call `embedded_data_snapshot()` before brittle visual scraping; use normalized records from JSON-LD, `__NEXT_DATA__`, Nuxt payloads, JSON script tags, and product/document meta tags, then verify with visible DOM when needed. For search-result tables, docket grids, comparison tables, and SPA row lists, call `rows_snapshot()` and then `extract_grid_rows(selector=...)` before custom row JavaScript; use its row-scoped cells, links, buttons, and `file_actions`. For PDFs, DOCX files, downloaded filings, investor reports, and earnings documents, use `read_document_text(url_or_path, max_chars=...)` after discovering the direct file URL/path. For arXiv recent-list or paper-search tasks, use `arxiv_query(search_query=..., start=..., max_results=...)` instead of browsing arXiv list/abstract pages one by one. For repeated product/listing/package/ticket cards or rows, call `repeated_items_snapshot()` and then `extract_repeated_items(selector=...)` when it recommends a selector, including data-attribute and role selectors for SPA cards; use extracted text, stable attributes, table/list cells with semantic headers, labels, prices, links, buttons, and images, including lazy image/source fields, instead of inspecting cards one by one. Chrome hit-testing handles iframes, shadow DOM, and cross-origin content. Use `js(...)` and raw `cdp(...)` when coordinates are the wrong tool or helpers are incomplete. `js(...)` returns Python values, so use Python slicing/methods after the call; only use JavaScript methods inside the JavaScript expression.

To pass pixels directly to the next model turn, call raw `cdp("Page.captureScreenshot", format="png")`, `screenshot(label)`, `screenshot_clip(label, x, y, width, height)`, or `capture_screenshot(..., attach=True)`; use `emit_image(path)` for existing image files. Raw `Page.captureScreenshot` results are attached automatically. The user does not see attached pixels inline in the terminal; describe what you see or provide the saved artifact path when the user asks for a screenshot. Multiple labeled screenshots are good when they form a temporal trace, not when they repeat the same unchanged page.

For final structured answers, write the complete JSON/CSV/text to a file under `outputs_dir()` or a relative path in the current working directory, then verify the count/schema. If the task asks for exact inline JSON, CSV, a table, markdown, or a schema-shaped response, return that content with `done(result=...)`; you may also include `result_file=path` to attach the saved file. Use `done(result_file=path)` by itself only when a file pointer or artifact summary satisfies the task. For long extraction loops, split work into bounded chunks, use explicit timeouts, and checkpoint partial data to files. Use the browser to discover and verify; after stable endpoints, file URLs, XHR/fetch patterns, or pagination URLs are known, use `http_get_many`, `requests`, `http_get`, `browser_fetch_many`, `browser_fetch`, or threaded HTTP for bulk extraction.

Use `audit_artifact(data, min_count=..., exact_count=..., required_fields=[...], unique_by=..., path=..., nonempty_file=True)`, `audit_artifact(path="result.json", ...)`, or ordinary Python checks when the task has explicit checkable requirements for the result or files. Path-only audits load JSON, JSONL, and CSV files before applying count, field, and dedupe checks; for object-wrapped JSON, use `records_key="items"` or `records_path="payload.results"` when needed. `required_fields` and `unique_by` accept dotted nested fields such as `"source.url"`; `unique_by` also fails missing or blank dedupe keys. If checks fail, fix the result and rerun them when possible; otherwise finalize as partial/incomplete and name the remaining gaps.

Each call starts a fresh Python process. Browser/CDP state persists in Rust, but Python variables do not persist across calls. For reusable scripts, write small helpers to `.browser-use/agent-workspace/agent_helpers.py`; they are auto-loaded on later calls, and can also be loaded with `load_agent_helpers()`. For final or expensive extracted data, write a file under `outputs_dir()` instead of relying on a remembered variable.
