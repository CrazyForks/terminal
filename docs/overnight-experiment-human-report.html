What Got Better

final answersoutput shaperunner reliabilitysource honesty

  • Final answers no longer commonly become Done., Need final done..., or preview-only snippets.
  • If a ready persisted final answer exists at max turns, the harness can finish with it instead of failing the run.
  • Large JSON tasks are more likely to return the requested JSON object/list instead of a path-only summary.
  • Source-scope fallbacks are more often marked partial instead of being counted as complete.
  • Cloud-browser concurrency itself is healthy. The bad local-Chrome behavior was avoided in the later runs.

What Regressed

Yes, there were regressions. The important one: after 926c0b0, real_v8 dropped from 73/23/4 to 67/29/4. The next semantic/finalization change recovered most of it to 71/26/3, but not all the way back.

  • Dynamic offer pages: tasks 47, 49, 50 regressed to partial because the agent missed lazy visual content or stopped before hidden offer families were exhausted.
  • Stateful forms: tasks 21 and 40 still fail/partial because the model does not prove dates/address/filter state is actually applied.
  • Coverage/frontier: some all/load-more/sitemap tasks produce counts/samples but not complete frontier evidence.
  • Semantic field values: the model sometimes guesses or normalizes fields, e.g. contract length/category/operator ID, instead of requiring source proof.

Did We Overfit?

Short answer: mostly no, but there is moderate prompt-overfitting risk.

  • Not overfit: no deterministic benchmark-answer checks were added. The code changes are general: timeouts, persisted final-answer handling, audit readiness, source evidence, visual artifact validity.
  • Some risk: prompt wording was shaped by recurring real_v8/real_v14 failures, so it may be too heavy around audits/finalization and could spend more turns or produce more partial answers.
  • Good bitter-pill direction: the branch mostly teaches the model to verify and represent evidence instead of adding task-specific validators.
  • Watch out: adding too many audit rules can make the agent conservative. That is better than wrong-complete answers, but it may lower runner pass rate or strict pass rate on tasks where enough data exists but the audit burden distracts the model.

Current Failure Modes

Interactive state not proven 21, 40 The page looks relevant, but selected dates/address/filter may not be accepted or bound to results. Require state proof in URL/page copy/selected controls/network response before extraction.
Dynamic frontier not exhausted 27, 49, 50, 63 The model stops at visible cards or samples without proving tabs, hidden sections, pages, or load-more are exhausted. Frontier artifact: page/API log, before/after counts, no-next/no-new evidence.
Visual/lazy content missed 47, v14 6 DOM extraction says content absent, but the information is visually rendered, lazy-loaded, or in screenshots/media. Use screenshots/OCR or saved HTML/media checks before declaring required visual sections absent.
